# Økt 1-2 — Layout (Flexbox/Grid) + Intro JavaScript

---

## Mål for økten

- Bruke Flexbox og/eller Grid til enkel layout
- Forstå hva JavaScript er og hva det kan gjøre
- Skrive og kalle en funksjon
- Endre siden med `getElementById` + `innerHTML`
- Koble en knapp til en funksjon med `onclick`

---

## Del 1 — Layout med CSS (~30 %)

### Hva er layout?

Vis problemet: elementer stablet loddrett som standard. Hva om vi vil ha dem ved siden av hverandre?

### Demo 1 — Flexbox

```css
.rad {
  display: flex;
  gap: 16px;
}
```

```html
<div class="rad">
  <div>Boks 1</div>
  <div>Boks 2</div>
  <div>Boks 3</div>
</div>
```

utforske selv:

- `justify-content: center` / `space-between`
- `align-items: center`
- `flex-direction: column`

### Demo 2 — Grid

```css
.rutenett {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  gap: 16px;
}
```

```html
<div class="rutenett">
  <div>Celle 1</div>
  <div>Celle 2</div>
  <div>Celle 3</div>
  <div>Celle 4</div>
  <div>Celle 5</div>
  <div>Celle 6</div>
</div>
```

### Selvutforsking

eksperimenter i DevTools og prøv egne verdier. Trenger ikke å forstå alt — se _hva som er mulig_.

> Ressurs å vise frem: https://css-tricks.com/snippets/css/a-guide-to-flexbox/, https://css-tricks.com/complete-guide-css-grid-layout/

## Del 2 — Intro JavaScript (~60 %)

### Hva er JavaScript?

> HTML = kroppen. CSS = klærne. JavaScript = musklene.

JavaScript lar siden _reagere_ på brukeren.

### Skill 1 — Funksjon

Skriv dette i konsollen (F12):

```js
function siHei() {
  console.log("Hei!");
}

siHei();
```

Poeng:

- `function` er nøkkelordet
- Navnet velger vi selv
- Parentesene `()` betyr "kjør nå"
- Koden i `{ }` kjøres når funksjonen kalles

### Skill 2 — `getElementById` og `innerHTML`

```html
<p id="output">Her er teksten</p>
```

```js
document.getElementById("output").innerHTML = "JavaScript endret dette!";
```

Prøv i konsollen på siden.

Poeng:

- `document` er hele siden
- `getElementById("output")` finner elementet med den ID-en
- `.innerHTML =` bytter ut innholdet

### Skill 3 — `onclick`

```html
<button onclick="siHei()">Klikk meg</button>
<p id="output"></p>
```

```js
function siHei() {
  document.getElementById("output").innerHTML = "Du klikket!";
}
```

### Demo 3 — Alle tre samlet (bygg steg for steg)

```html
<!DOCTYPE html>
<html lang="no">
  <head>
    <meta charset="UTF-8" />
    <title>Hilsen</title>
  </head>
  <body>
    <input id="navnInput" type="text" placeholder="Skriv navn" />
    <button onclick="visHilsen()">Si hei</button>
    <p id="output"></p>

    <script>
      function visHilsen() {
        let navn = document.getElementById("navnInput").value;
        document.getElementById("output").innerHTML = "Hei, " + navn + "!";
      }
    </script>
  </body>
</html>
```

### Oppgave

En liten leilighet har en gang, et soverom, et bad og en stue. Lag en nettside som starter med å si "Du er utenfor en leilighet." og har en knapp for å gå inn. Når man trykker på knappen, skal siden si "Du er i gangen" - og det skal være knapper for å gå ut, gå inn i stua, gå inn på badet og gå inn på kjøkkenet. Fra alle de andre rommene skal man kun få gå tilbake til gangen - med unntak av at man kan gå fra kjøkkenet til stua og omvendt. Lag ferdig så man kan gå til alle rom. Til dette trenger du kun onclick og enkle JavaScript-funksjoner som endrer HTML til en div.

---

## Vanlige feil

| Feil                   | Årsak                                             | Løsning                  |
| ---------------------- | ------------------------------------------------- | ------------------------ |
| Knappen gjør ingenting | `onclick="visHilsen"` uten `()`                   | Legg til parenteser      |
| Element ikke funnet    | ID stemmer ikke                                   | Sammenlign tegn for tegn |
| CSS layout virker ikke | Mangler `display: flex/grid` på foreldreelementet | Inspiser i DevTools      |

---

## Tips

- Hold JavaScript-demoen enkel — tre ting, ikke mer
- Les feilmeldingen i konsollen høyt: "Hva tror du den sier?"
- CSS-layout: kopier og endre verdier, ikke prøv å forstå alt
