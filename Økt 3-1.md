## Mål for økten
- Forstå hva en returverdi er
- Skrive funksjoner som returnerer HTML-strenger
- Bruke funksjoner som byggeklosser (komponenter)
- Se fordelen med DRY: Don't Repeat Yourself

---

## Del 1 — Hva er en returverdi? (~25 %)

### Demo 1 — Funksjon uten returverdi (repetisjon)

```js
function siHei(navn) {
    document.getElementById("output").innerHTML = "Hei, " + navn + "!";
}
```

Problemet: funksjonen gjør jobben selv, men resultatet kan ikke gjenbrukes.

### Demo 2 — Funksjon med returverdi

```js
function lagHilsen(navn) {
    return "Hei, " + navn + "!";
}

let tekst = lagHilsen("Kari");
console.log(tekst);  // "Hei, Kari!"
```

Poeng:
- `return` sender en verdi *tilbake* til den som kalte funksjonen
- Funksjonen produserer noe — vi bestemmer selv hva vi gjør med det
- En funksjon med `return` er som en maskin: du putter noe inn, du får noe ut

## Del 2 — Funksjoner som HTML-komponenter (~50 %)

### Demo 3 — Enkel komponentfunksjon

```js
function lagKort(tittel, tekst) {
    return "<div class='kort'>" +
               "<h2>" + tittel + "</h2>" +
               "<p>" + tekst + "</p>" +
           "</div>";
}
```

Vi *bruker* komponenten:
```js
document.getElementById("output").innerHTML =
    lagKort("HTML", "Struktur") +
    lagKort("CSS", "Utseende") +
    lagKort("JavaScript", "Interaktivitet");
```

Poeng:
- Funksjonen returnerer en HTML-streng
- Vi kan kalle den mange ganger og sette sammen resultatet
- Vil vi endre kortets utseende → endrer vi funksjonen *ett sted*

### Demo 4 — Komponenter som bygger på hverandre

```js
function lagBilde(src, alt) {
    return "<img src='" + src + "' alt='" + alt + "'>";
}

function lagProfilkort(navn, bildeUrl) {
    return "<div class='profilkort'>" +
               lagBilde(bildeUrl, navn) +
               "<h3>" + navn + "</h3>" +
           "</div>";
}

document.getElementById("output").innerHTML =
    lagProfilkort("Kari", "https://via.placeholder.com/100") +
    lagProfilkort("Ole", "https://via.placeholder.com/100");
```

Poeng:
- En funksjon kan kalle en annen funksjon
- Komplekse komponenter bygges av enkle deler
- Dette er samme tankegang som i ekte frontend-utvikling (React, Vue osv.)

## Del 3 — DRY i praksis (~15 %)

### Uten funksjoner
```html
<div class="kort"><h2>HTML</h2><p>Struktur</p></div>
<div class="kort"><h2>CSS</h2><p>Utseende</p></div>
<div class="kort"><h2>JavaScript</h2><p>Interaktivitet</p></div>
```

Problemet: vil vi legge til et ikon i hvert kort må vi endre tre steder.

### Med funksjoner
```js
function lagKort(tittel, tekst) {
    return "<div class='kort'><span>📌</span><h2>" + tittel + "</h2><p>" + tekst + "</p></div>";
}
```

Vi endrer *ett sted* — alle kortene oppdateres.

> "Don't Repeat Yourself" — skriv ting én gang, bruk det mange ganger.

## Vanlige feil

| Feil | Årsak | Løsning |
|---|---|---|
| Funksjonen returnerer `undefined` | Glemte `return` | Sjekk at `return` er inne i funksjonen |
| HTML vises som tekst | Enkle hermetegn krasjer med ytre hermetegn | Bruk `'` inne i HTML-strengen |
| Ingenting vises på siden | Resultatet av funksjonen brukes ikke | Husk `innerHTML =` der funksjonen kalles |

## Tips
- Start med Demo 3 alene — én komponent, tre kall, se resultatet
- Se siden *før og etter* at funksjonen legger til ett nytt element
- Knytt til noe du kjenner: "Det er som en mal du fyller inn i"
