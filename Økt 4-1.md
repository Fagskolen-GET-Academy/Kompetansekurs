# Økt 4-1 — Intro MVC: veldig enkelt
**Dato:** mandag 11. mai 2026

---

## Mål for økten
- Forstå *problemet* MVC løser
- Skille mellom data (model), visning (view) og logikk (controller)
- Lese og oppdatere en enkel model-variabel
- Ha en `render()`-funksjon som alltid tegner modellen

---

## Del 1 — Problemet med "kaos-kode" (~20 %)

### Vis dette eksemplet — hva er galt?

```html
<button onclick="document.getElementById('t').innerHTML = Number(document.getElementById('t').innerHTML) + 1">
    Øk
</button>
<p id="t">0</p>
```

Spørsmål:
> "Hva er verdien av telleren akkurat nå?"

Svaret: vi vet ikke uten å lese siden. Dataen *lever i HTML-en*, ikke i koden.

Problemet skalerer:
- Hva om vi har ti elementer som skal vise samme data?
- Hva om vi vil sjekke verdien i en if-setning?

---

## Del 2 — MVC-tankegang (~30 %)

### De tre delene

```
MODEL      — dataen. Sannheten om hva som skjer i appen.
VIEW       — det brukeren ser. Tegnes alltid fra modellen.
CONTROLLER — det som skjer når brukeren gjør noe. Oppdaterer modellen, kaller render.
```

### Regler vi følger

1. Data bor i modellen — ikke i HTML
2. `render()` leser alltid fra modellen og oppdaterer HTML
3. Brukerhendelser (onclick) kaller controllerfunksjoner
4. Controllerfunksjoner endrer modellen og kaller `render()`

### Demo 1 — Teller i MVC-stil

```html
<!DOCTYPE html>
<html lang="no">
<head>
    <meta charset="UTF-8">
    <title>Teller – MVC</title>
</head>
<body>
    <h1 id="teller">0</h1>
    <button onclick="ok()">Øk</button>
    <button onclick="nullstill()">Nullstill</button>

    <script>
        // MODEL
        let model = 0;

        // VIEW
        function render() {
            document.getElementById("teller").innerHTML = model;
        }

        // CONTROLLER
        function ok() {
            model += 1;
            render();
        }

        function nullstill() {
            model = 0;
            render();
        }

        render(); // tegn starttilstand
    </script>
</body>
</html>
```

Gå gjennom koden del for del:
- **MODEL**: `let model = 0` — her bor dataen
- **VIEW**: `render()` — leser modellen og oppdaterer HTML
- **CONTROLLER**: `ok()` og `nullstill()` — endrer modellen og kaller `render()`

## Del 3 — Utvidelse: model med mer informasjon (~30 %)

Når appen vokser, trenger modellen mer enn én verdi.

```js
// MODEL — samle all data ett sted
let model = 0;  // holder det enkelt foreløpig
```

Vis at render() alltid kan stole på at modellen er sannheten:

```js
function render() {
    document.getElementById("teller").innerHTML = model;

    if (model >= 10) {
        document.getElementById("melding").innerHTML = "Du er over 10!";
    } else {
        document.getElementById("melding").innerHTML = "";
    }
}
```

Poeng:
- render() kan gjøre mange ting — alt basert på samme model
- Vi slipper å oppdatere HTML manuelt fra mange steder

---

## Del 4 — Oppsummering og mønster (~10 %)

```
onclick → controller → oppdater model → render() → HTML oppdateres
```

Dette er flyten i alle MVC-apper, uansett størrelse.

> "Alt dere har lært så langt — funksjoner, variabler, if — brukes nå sammen."

## Vanlige feil

| Feil | Årsak | Løsning |
|---|---|---|
| Siden oppdateres ikke | Glemte å kalle `render()` i controllerfunksjonen | Alle controllerfunksjoner skal avslutte med `render()` |
| Modellen er ikke oppdatert | Endrer HTML i stedet for model | Endre alltid `model`, aldri HTML direkte |
| `render()` viser feil | Leser fra HTML i stedet for model | render() skal *alltid* lese fra `model` |

---

## Tips
- Start med å vise telleren *uten* MVC, deretter *med* — se forskjellen
- MVC er en tankegang, ikke magi — bruk tid på "hvorfor" før "hvordan"
- Kommentarene `// MODEL`, `// VIEW`, `// CONTROLLER` i koden er pedagogisk viktige — ikke fjern dem
