# Økt 3-2 — Full operatorpakke + null og undefined
**Dato:** onsdag 6. mai 2026
**Tid:** ca. 1,5–3 timer

---

## Mål for økten
- Bruke tilordningsoperatorer som snarveier
- Kombinere sammenligninger med logiske operatorer (`&&`, `||`, `!`)
- Forstå hva `null` og `undefined` er og når de dukker opp

---

## Del 1 — Tilordningsoperatorer (~20 %)

Vi kjenner `=` fra før. Nå ser vi på snarveiene:

```js
let poeng = 10;

poeng = poeng + 5;   // gammel måte
poeng += 5;          // kortform — samme resultat

poeng -= 3;          // trekk fra 3
poeng *= 2;          // gang med 2
poeng /= 4;          // del på 4
```

### Demo 1 — Teller med kortform

```js
let antall = 0;

function ok() {
    antall += 1;
    document.getElementById("teller").innerHTML = antall;
}

function trekkFra() {
    antall -= 1;
    document.getElementById("teller").innerHTML = antall;
}
```

## Del 2 — Logiske operatorer (~35 %)

### `&&` — OG: begge må være sanne

```js
let alder = 20;
let harBillett = true;

if (alder >= 18 && harBillett) {
    console.log("Du kommer inn!");
} else {
    console.log("Nei.");
}
```

### `||` — ELLER: minst én må være sann

```js
let erStudent = false;
let erPensjonist = true;

if (erStudent || erPensjonist) {
    console.log("Du får rabatt.");
} else {
    console.log("Ingen rabatt.");
}
```

### `!` — IKKE: snur sant/usant

```js
let erPålogget = false;

if (!erPålogget) {
    console.log("Vennligst logg inn.");
}
```

### Demo 2 — Validering av inputfelt

```html
<input id="brukernavn" placeholder="Brukernavn (min 3 tegn)">
<input id="passord" placeholder="Passord (min 6 tegn)">
<button onclick="sjekkSkjema()">Logg inn</button>
<p id="melding"></p>

<script>
    function sjekkSkjema() {
        let brukernavn = document.getElementById("brukernavn").value;
        let passord = document.getElementById("passord").value;

        if (brukernavn.length >= 3 && passord.length >= 6) {
            document.getElementById("melding").innerHTML = "Innlogget!";
        } else {
            document.getElementById("melding").innerHTML = "Fyll ut skjemaet riktig.";
        }
    }
</script>
```

## Del 3 — null og undefined (~35 %)

### Hva er `undefined`?

`undefined` betyr at noe *eksisterer*, men ikke har fått en verdi ennå.

```js
let navn;
console.log(navn);   // undefined
```

Oppstår også når:
- Vi prøver å lese et element som ikke finnes på siden
- En funksjon mangler `return`

### Hva er `null`?

`null` betyr *bevisst ingenting* — vi har satt det til null med vilje.

```js
let valgtFarge = null;

// Senere i koden:
valgtFarge = "blå";
```

### Demo 3 — Sjekk om input er tom

```js
function visNavn() {
    let navn = document.getElementById("navnInput").value;

    if (navn === "" || navn === null || navn === undefined) {
        document.getElementById("output").innerHTML = "Du skrev ikke inn et navn.";
    } else {
        document.getElementById("output").innerHTML = "Hei, " + navn + "!";
    }
}
```

Kortere variant:
```js
if (!navn) {
    // Kjøres hvis navn er tom streng, null eller undefined
}
```

Poeng:
- `!navn` er `true` hvis `navn` er `""`, `null` eller `undefined`
- Dette er et veldig vanlig mønster i JavaScript

## Vanlige feil

| Feil | Årsak | Løsning |
|---|---|---|
| `&&` gir alltid `false` | Den ene betingelsen er alltid usann | Test betingelsene separat i konsollen |
| `=` i stedet for `===` | Tilordning vs. sammenligning | `=` setter, `===` sjekker |
| `undefined` dukker opp uventet | Elementet finnes ikke eller ID er feil | Sjekk ID og at elementet er i HTML |

---

## Tips for de som henger etter
- Start med `&&` alene — det er det mest brukte
- `null`/`undefined`-seksjonen kan gjøres rask og praktisk: "Dere vil se disse, og nå vet dere hva de betyr"
- Bruk konsollen aktivt: `let x; console.log(x)` — la dem se det selv
